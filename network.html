Network Architecture

Astar Network is built on Substrate, the modular blockchain framework developed by Parity Technologies. Its runtime includes the Frontier pallet stack, which adds a full Ethereum-compatible execution environment on top of Substrate's WASM runtime. This dual-layer architecture enables Astar to serve both Ethereum and Polkadot developers with a single chain.

Substrate + Frontier EVM Layer

Astar's EVM implementation uses Parity's Frontier pallet, which maps Ethereum account addresses and private keys to Substrate accounts using a unified address derivation scheme. The Frontier EVM processes transactions in a sandboxed environment within the Substrate block, producing Ethereum-compatible receipts and logs that are queryable via standard eth_* methods.

Ethereum RPC Surface

  • eth_* — Block, transaction, log, filter queries
  • net_* — Network peer and version info
  • web3_* — Client version and SHA3
  • eth_subscribe — WebSocket push subscriptions

Substrate RPC Surface

  • chain_* — Block headers, hashes, finality
  • state_* — Runtime storage, metadata
  • system_* — Node health, peers, sync state
  • author_* — Extrinsic submission, pending pool

Asynchronous Backing

Asynchronous Backing is a Polkadot 2.0 upgrade that decouples block authoring from the relay chain's synchronous slot mechanism. Astar benefits from this with approximately 6-second block times on mainnet (halved from the original ~12 seconds), doubling throughput and reducing transaction confirmation latency without any changes to the EVM or Substrate API surface.

Agile Coretime

Agile Coretime replaces the traditional parachain slot auction model with a flexible execution core marketplace. Astar can acquire bulk coretime for predictable block production or purchase on-demand coretime for burst workloads. This model enables more efficient use of Polkadot's shared security resources and allows Astar to scale block production dynamically based on network demand and ecosystem activity.
