Running an Astar Node

Astar Network supports several node types: collator nodes (block producers), archive nodes (full historical state), and light clients. Collators participate in parachain block production and receive rewards. Archive nodes are used for indexers, explorers, and applications requiring historical state queries. Running your own node provides maximum decentralisation and eliminates reliance on third-party RPC providers.

Node Types

Collator Node

Produces parachain blocks, collects extrinsics from the mempool, and submits PoV (Proof of Validity) blocks to the Polkadot relay chain validators. Collators must be whitelisted or bond a collation deposit (varies by network). They receive block rewards from the parachain.

Archive Node

Retains full historical state (all blocks, all storage root snapshots). Required for state_getStorage calls at historical blocks. Storage requirements are significantly higher — plan for 4 TB+ NVMe for full Astar history as the chain matures.

Full Node (RPC)

Validates all blocks and maintains current state without full history. Suitable for serving RPC requests for current-state queries, sending transactions, and subscribing to new heads. This is the standard node type for self-hosted RPC infrastructure.

Installation

Download the latest Astar binary from the official GitHub releases page and configure systemd for process management:

# Download latest release
RELEASE=$(curl -s https://api.github.com/repos/AstarNetwork/Astar/releases/latest   | grep tag_name | cut -d '"' -f4)
wget "https://github.com/AstarNetwork/Astar/releases/download/${RELEASE}/astar-collator-${RELEASE}-ubuntu-x86_64.tar.gz"
tar -xzf astar-collator-*.tar.gz
sudo mv astar-collator /usr/local/bin/
sudo chmod +x /usr/local/bin/astar-collator

# Verify installation
astar-collator --version

# Start as full RPC node
astar-collator   --chain astar   --rpc-port 9944   --rpc-cors all   --pruning archive   --base-path /var/lib/astar   --   --chain polkadot   --sync warp

Monitoring

Astar nodes expose a Prometheus metrics endpoint on port 9615 by default (enable with --prometheus-port 9615). Use Grafana with the official Astar dashboard to monitor block lag, peer count, RPC request rates, and memory usage. Set alerts for block production gaps (>30s), peer count drops below 10, and disk I/O saturation.
